Overview

What ships today.

One stable Cosmos DB skill, a static GitHub Pages site, local-first Vally evaluation, and install surfaces for Copilot, Claude Code, Cursor, Codex, and Gemini CLI.

Primary skill cosmosdb-best-practices

Stable guidance for Azure Cosmos DB application design and implementation.

Evaluation model Local-first with Vally

Run evals locally to sanity-check changes without wiring the content scoring into CI.

Docs target GitHub Pages in docs/

The website ships as static HTML, CSS, and JS with no backend dependency.

Why teams use it

It narrows the gap between generic code generation and Cosmos DB-specific engineering judgment.

High-impact rules first

Critical guidance for partition strategy, data modeling, and query tuning is prioritized so the agent makes architectural decisions before code generation drifts into expensive rework.

Deterministic iteration loops

Scenario contracts, harness templates, and improvements logs make regressions visible and keep rule changes tied to concrete failures.

Portable skill packaging

The kit follows the Agent Skills format so the same knowledge can be reused across Copilot, Claude Code, Cursor, Codex, Gemini, and adjacent toolchains.

Install

Start with a universal install, then validate and build locally when you contribute.

Fastest install

npx skills add AzureCosmosDB/cosmosdb-agent-kit

Drops the skill catalog into the compatible agent host you are using.

All supported harnesses

apm install AzureCosmosDB/cosmosdb-agent-kit

Installs across GitHub Copilot, Claude Code, Cursor, Codex, and Gemini in one command.

Local contributor flow

npm install
npm run validate
npm run build

Use this after changing rules, scenarios, or generated skill outputs.

Workflow

Contribute with an evaluation loop that stays concrete.

1. Pick a scenario

Use the templates in testing-v2/scenarios/ to define a scenario brief, contract, and iteration target.

2. Run an iteration

Create an iteration folder, generate or revise implementation code, and keep outcomes tied to the scenario.

3. Evaluate and improve

Use Vally for local skill checks, classify failures, adjust rules, rebuild AGENTS output, and record the improvement.

Local evaluation vally run evals/cosmosdb-best-practices/eval.yaml -v
Skill readiness vally check skills/cosmosdb-best-practices
